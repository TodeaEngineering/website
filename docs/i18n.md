# Internationalization (i18n)

## Supported Locales

| Code | Language            |
| ---- | ------------------- |
| `en` | English (default)   |
| `ko` | Korean (한국어)      |
| `ja` | Japanese (日本語)    |
| `zh` | Chinese (中文)       |

## How It Works

The site uses [next-intl](https://next-intl-docs.vercel.app/) with the Next.js App Router.

1. **Locale routing** is defined in `src/i18n/routing.ts`. To add a new locale, add its code to the `locales` array.
2. **Translation files** live in `src/messages/{locale}.json`. Each file has the same keys organized by component name (e.g., `Nav`, `Hero`, `Services`).
3. **Middleware** (`src/middleware.ts`) detects the user's locale from the URL prefix and serves the correct translations.
4. **Components** access translations via the `useTranslations` hook (client) or `getTranslations` (server).

## Adding a New Locale

1. Add the locale code to `src/i18n/routing.ts`:
   ```ts
   locales: ['en', 'ko', 'ja', 'zh', 'de'],
   ```
2. Create `src/messages/de.json` with all the same keys as `en.json`, translated.
3. Update the `matcher` in `src/middleware.ts`:
   ```ts
   matcher: ['/', '/(en|ko|ja|zh|de)/:path*'],
   ```
4. Add the language option to the `langs` array in `src/components/Nav.tsx`.

## Adding New Translation Keys

When adding a new component or section:

1. Add keys under a new namespace in all locale files (e.g., `"MyComponent": { "title": "..." }`).
2. Use `useTranslations('MyComponent')` in the component.
